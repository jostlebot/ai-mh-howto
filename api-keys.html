Understanding API Keys

What is an API Key?

Think of an API key like a special password that lets your projects talk to AI services.

When you use Claude.ai in your browser, you're logged in with your email. But when you want to build something that uses Claude — like a custom app or automation — you need an API key to prove it's really you.

Analogy: Your email login is like walking into a store and showing your face. An API key is like giving a trusted assistant your credit card to shop on your behalf.

Step 1: Get Your Anthropic API Key

  1. Go to console.anthropic.com
  2. Log in (or create an account if needed)
  3. Click on "API Keys" in the left sidebar
  4. Click "Create Key"
  5. Give it a name like "my-first-key"
  6. Copy the key immediately — you won't be able to see it again
Important: Your API key will look something like:
sk-ant-api03-xxxxxx...
Keep this secret. Anyone with this key can use your account and you'll be charged.

Step 2: Store Your Key Safely

Never put API keys directly in code that you share. Here's the safe way:

On Mac:

  1. Open Terminal
  2. Type this command (replace YOUR_KEY with your actual key):
echo 'export ANTHROPIC_API_KEY="YOUR_KEY"' >> ~/.zshrc
  1. Then run:
source ~/.zshrc

Now your key is stored securely and available whenever you need it.

What this does:

It saves your key as an "environment variable" — a piece of information your computer remembers but doesn't show in your files.

Step 3: Verify It Works

Open a new Terminal window and type:

echo $ANTHROPIC_API_KEY

You should see your key printed out. If you see nothing, try closing Terminal and opening it again.

Understanding API Costs

When you use Claude through the API, you pay for what you use. It's measured in "tokens" — roughly:

  • 1 token ≈ 4 characters of text
  • A typical conversation might use 1,000-5,000 tokens
  • Current pricing is fractions of a penny per 1,000 tokens

For learning and small projects, expect to spend $5-20/month at most. You can set spending limits in your Anthropic console.

Safety Rules

DO:

  • Store keys in environment variables
  • Set spending limits on your account
  • Create separate keys for different projects
  • Delete keys you're no longer using

DON'T:

  • Put keys in files you share publicly
  • Share keys with others (they should get their own)
  • Commit keys to GitHub
  • Post keys in screenshots or videos
