Setting Up GitHub

Do You Need This?

Yes, if you want to:

  • Deploy websites (like this one) for free
  • Back up your projects automatically
  • Track changes to your work over time
  • Share code or collaborate with others
  • Let Claude Code deploy things directly to the web

You can skip this if: You only want to use Claude Code locally for content generation and won't be deploying sites.

What is GitHub?

GitHub is where developers store and share code. Think of it as Google Drive, but specifically designed for code projects. It's free for everything we'll do.

When Claude Code creates a website for you, GitHub is how we put it on the internet.

Step 1: Create a GitHub Account

  1. Go to github.com
  2. Click "Sign up"
  3. Use your email, create a username, set a password
  4. Verify your email when they send the confirmation

That's it — you have a GitHub account.

Step 2: Install the GitHub CLI

The GitHub CLI lets Claude Code talk directly to GitHub — creating repositories, deploying sites, all from the terminal.

On Mac:

Open Terminal and run:

brew install gh
Don't have Homebrew? Install it first by running this in Terminal: 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Then try the brew install gh command again.

On Windows:

winget install --id GitHub.cli

Step 3: Log In to GitHub CLI

In Terminal, run:

gh auth login

You'll see prompts. Choose these options:

  1. Where do you use GitHub? → GitHub.com
  2. Preferred protocol? → HTTPS
  3. Authenticate? → Login with a web browser

It will give you a code and open your browser. Paste the code, authorize, and you're connected.

Step 4: Verify It Works

Run this command:

gh auth status

You should see something like:

github.com
  ✓ Logged in to github.com account YourUsername

How Claude Code Uses GitHub

Once you're set up, Claude Code can:

  • Create repositories: "Create a new repo called my-project"
  • Deploy sites: "Deploy this to GitHub Pages"
  • Push updates: "Push these changes to GitHub"
  • Check status: "Is my site live?"

You just ask in plain English. Claude handles the commands.

Example: Deploying a Site

Here's what a real conversation looks like:

You: Create a simple portfolio site for me and deploy it

Claude: I'll create an index.html with your info... [creates files]

Claude: Now I'll create a GitHub repository and enable GitHub Pages... [runs commands]

Claude: Done! Your site is live at: https://yourusername.github.io/portfolio/

That's it. From idea to live website in one conversation.

Understanding the Basics

A few terms you'll hear:

  • Repository (repo): A project folder stored on GitHub
  • Commit: Saving a snapshot of your changes
  • Push: Sending your changes to GitHub
  • GitHub Pages: Free website hosting from GitHub

You don't need to memorize commands. Just tell Claude what you want, and it will handle the git operations.

Troubleshooting

If brew isn't found: You need to install Homebrew first (see Step 2).
If authentication fails: Try running gh auth login again and make sure you complete the browser step.
If GitHub Pages isn't working: It can take 1-2 minutes to build. Refresh after waiting.
